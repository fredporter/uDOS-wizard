# v2.0.2 Runtime Service Consumption

`uDOS-wizard` consumes the `uDOS-core` runtime-service catalog to explain how
its orchestration surfaces attach to the shared platform contracts.

## Status Extensions

`/orchestration/status` now reports:

- `foundation_version`
- `runtime_services`
- `runtime_service_source`

The shared dispatch path is:

- `/orchestration/dispatch?task=<task>&mode=<mode>&surface=<surface>`

## Consumption Rule

- `uDOS-core` owns the runtime-service contract definitions
- `uDOS-wizard` reports which of those services it consumes from
  `uDOS-core/contracts/runtime-services.json`
- orchestration status may describe usage, but it does not replace the Core
  manifest
