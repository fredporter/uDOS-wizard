# uDOS-wizard Getting Started

1. Read `docs/boundary.md` before adding provider integrations.
2. Read `docs/first-launch-quickstart.md` for the current runnable service and GUI path.
3. Read `docs/v2.0.3-port-binding.md` for the current host/port behavior.
4. Read `docs/first-launch-quickstart.md#dual-service-workflow-loop` before changing Wizard/uHOME handoff behavior.
5. Keep network adapters under `services/runtime/providers/`.
6. Route budget policy through `services/runtime/budgeting/`.
7. Add tests for every public adapter contract.
8. Run `scripts/run-wizard-checks.sh` before promotion.
9. Use `examples/basic-wizard-session.md` for the first local activation walkthrough.
