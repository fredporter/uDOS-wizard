# v2.0.4 Wizard Sibling Route Set

## Purpose

Define which Wizard HTTP routes sibling repos may call directly during the
`v2.0.4` networking boundary lock.

The rule is narrow: sibling repos may call Wizard-owned routes that expose
Wizard-owned orchestration, workflow, grid-consumption, and operator-facing
inspection surfaces. Sibling repos must not absorb provider routing, secret
mutation, or bridge policy just because they can call Wizard.

## Direct-Call Route Set

### `uDOS-shell`

Allowed now:

| Method | Route | Purpose | Status |
| --- | --- | --- | --- |
| `GET` | `/assist` | live online assist handoff from `#wizard assist topic:<task>` | active |
| `POST` | `/workflow/actions` | live workflow action handoff from `workflow action <workflow-id> <action>` | active |
| `POST` | `/workflow/handoff/automation-job` | live automation handoff from `automation queue <capability>` | active |
| `GET` | `/orchestration/status` | operator/status inspection | allowed |
| `GET` | `/orchestration/workflow-plan` | read-only workflow planning envelope | allowed |
| `GET` | `/grid/contracts/{name}` | inspect Grid-owned contracts through Wizard | allowed |
| `GET` | `/grid/seeds/{name}` | inspect Grid seed registries through Wizard | allowed |

Not allowed yet:

| Method | Route | Reason |
| --- | --- | --- |
| `POST` | `/orchestration/callback` | callback ownership stays inside Wizard-managed orchestration |
| `GET` | `/orchestration/result/{dispatch_id}` | result lookup remains Wizard-internal until dispatch lanes are locked |
| `POST` | `/workflow/reconcile/automation-result` | reconciliation belongs to Wizard + `uHOME-server` bridge ownership |

### `uHOME-server`

Allowed now:

| Method | Route | Purpose | Status |
| --- | --- | --- | --- |
| `GET` | `/workflow/state` | inspect active Wizard workflow state | allowed |
| `POST` | `/workflow/reconcile/automation-result` | hand back durable automation outcomes to Wizard | allowed |
| `POST` | `/workflow/reconcile/uhome-latest` | apply latest server automation result back to Wizard workflow state | allowed |

### `uHOME-empire`

Allowed now:

| Method | Route | Purpose | Status |
| --- | --- | --- | --- |
| `GET` | `/orchestration/status` | inspect Wizard transport/provider status | allowed |
| `GET` | `/orchestration/workflow-plan` | request read-only workflow plans for sync or provider work | allowed |
| `GET` | `/assist` | use Wizard provider routing for provider-backed assist lanes | allowed |

## Ownership Rule

- calling a Wizard route does not transfer route ownership
- calling a Wizard route does not transfer secret storage ownership
- calling a Wizard route does not let sibling repos define provider routing or
  retry policy on behalf of Wizard

If a sibling repo needs a new live route, add it here as part of the boundary
lock instead of quietly calling an internal Wizard endpoint.
