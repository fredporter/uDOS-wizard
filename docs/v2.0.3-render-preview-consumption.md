# v2.0.3 Render Preview Consumption

Wizard does not own the canonical render transform in v2.

Wizard does own the networked runtime surfaces that make shared renders usable
in operator flows:

- GUI previews
- browser GUI routes and operator-facing preview panels
- Thin GUI handoff payloads for downstream local runtimes
- publish orchestration jobs
- Beacon-facing handoff and preview status

## Consumption Rule

Wizard should consume:

- render contracts from `uDOS-core`
- prose preset and theme adapter ids from `uDOS-themes`

Wizard should expose:

- preview routes for `gui-preview`
- orchestration/status views for `web-prose` and `email-html`
- Beacon handoff data for nearby library surfaces that use shared render
  outputs

## GUI Alignment

Wizard browser GUI and downstream Thin GUI surfaces should consume the same
preview payloads and theme refs.

The difference between them is shell ownership and launch mode:

- `uDOS-wizard` owns the browser operator GUI
- `uHOME-server` owns Thin GUI and kiosk-style local presentation shells

Shared render/theme contracts remain outside both repos.
