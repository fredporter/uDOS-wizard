# uDOS-wizard Boundary

## Owns

- provider and API transport
- assist routing
- MCP bridging
- budgeting and autonomy controls
- Beacon access services
- workflow authority and guided advancement
- browser operator GUI contracts
- render preview and publish orchestration surfaces built on shared contracts

## Does Not Own

- canonical parsing rules
- canonical render contracts
- Thin GUI or kiosk runtime ownership
- always-on automation fulfillment
- always-on home runtime services
- Google or HubSpot sync business logic owned by `uDOS-empire`
- Apple-native desktop sync owned outside the public family

## v2 Runtime Rule

`uDOS-wizard` is the conscious workflow surface.

It may:

- start or configure workflows
- approve or reject gated actions
- request online or provider-backed assist
- dispatch background fulfillment work to `uHOME-server`

It should not be required to stay open for routine local execution once work
has been handed off to the always-on runtime.

Wizard should consume shared minimal workflow and automation artifacts from:

- `uDOS-core/contracts/workflow-state-contract.json`
- `uDOS-core/contracts/workflow-action-contract.json`
- `uDOS-core/contracts/automation-job-contract.json`
- `uDOS-core/contracts/automation-result-contract.json`
