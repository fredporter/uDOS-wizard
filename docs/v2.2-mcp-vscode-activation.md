# v2.2 MCP VS Code Activation

Status: active
Owner: uDOS-wizard
Binder: #binder/wizard-v2-2-mcp-vscode

## Purpose

Activate the first working Wizard-managed MCP flow that a local editor client
can discover and invoke.

This is the minimum Round B path:

- list live tools from Wizard
- invoke one real OK route tool through Wizard MCP
- keep provider policy, budget checks, and secret ownership inside Wizard

## Local Launch

From `uDOS-wizard`:

```bash
python3 -m wizard.main
```

Default local base URL:

- `http://127.0.0.1:8100`

## Working MCP Surfaces

- `GET /mcp/tools`
  - live tool listing
- `POST /mcp/tools/{tool_name}/invoke`
  - direct HTTP invoke path for local clients
- `POST /mcp`
  - JSON-RPC shim supporting `initialize`, `tools/list`, and `tools/call`

## First Working Tool

Tool: `ok.route`

Purpose:

- submit a managed OK routing request through Wizard budget/provider policy

Example HTTP invoke:

```bash
curl -s http://127.0.0.1:8100/mcp/tools/ok.route/invoke \
  -H 'content-type: application/json' \
  -d '{
    "task": "summarize this changelog",
    "task_class": "summarize",
    "allowed_budget_groups": ["tier0_free", "tier1_economy"]
  }'
```

Example JSON-RPC invoke:

```bash
curl -s http://127.0.0.1:8100/mcp \
  -H 'content-type: application/json' \
  -d '{
    "jsonrpc": "2.0",
    "id": "req-1",
    "method": "tools/call",
    "params": {
      "name": "ok.route",
      "arguments": {
        "task": "summarize this changelog",
        "task_class": "summarize",
        "allowed_budget_groups": ["tier0_free", "tier1_economy"]
      }
    }
  }'
```

## VS Code Notes

Use a local MCP-capable client or extension that can talk to HTTP JSON-RPC.
For the first Round B pass, point that client at:

- `POST http://127.0.0.1:8100/mcp`

Starter client profile:

- `mcp/vscode-http-client-profile.json`

Required request methods in this pass:

- `initialize`
- `tools/list`
- `tools/call`

## Boundary Guardrails

- Wizard remains the owner of provider routing and managed MCP policy
- Core remains the owner of deterministic local runtime semantics
- no provider credentials are exposed through MCP responses
- this pass is local-only and intentionally minimal
